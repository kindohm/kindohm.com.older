"So this one day I was using an oscillator..."


audiolib.js has been one of my favorite JavaScript libraries to work with so far. Being somewhat of a synthesizer and audio geek, I love using it to produce audio with code.

Aside from audio synthesis and generating samples, audiolib.js includes fft.js to perform Fast Fourier Transforms (FFT). If you're not familiar with what an FFT is, think of any graphical output you've seen in a car stereo or MP3 player that displays the frequency specturm of the music being played. FFT's are used to compute those displays. With audiolib.js, you can use FFT's to create your own similar display on an HTML5 canvas.

So how does this work exactly...

Basically in the audiolib.js callback where samples are generated you push the samples to an FFT object:

oscillator.generate(); 
sample = oscillator.getMix(); 
fft.pushSample(sample);

In the context of the entire audio callback it looks like this:

function audioCallback(buffer, channels) { 
  var i, n, sample, bufferLength = buffer.length;  
  for (i = 0; i < bufferLength; i += channels) {  
    oscillator.generate(); 
    sample = oscillator.getMix(); 
    fft.pushSample(sample);  
    for (n = 0; n < channelCount; n++) { 
      buffer[i + n] = sample; 
    } 
  } 
}

When this code starts running, the FFT fills up with thousands of samples and it starts to do some analysis. Luckily you don't need to know anything about the magical code in fft.js that does all that!

Now you just need to draw the FFT spectrum on a canvas, which basically just some simple x/y graphing:

var width, height, context;  
window.addEventListener('load', function() {  
  var canvas, button;  
  canvas = document.getElementById('canvas'); 
  context = canvas.getContext('2d'); 
  width = canvas.width; 
  height = canvas.height; 
  context.fillStyle = "#ff0000"; 
  context.lineWidth = 1;  
});  

function drawFFT () {  
  var length, count; 
  length = fft.spectrum.length / 8;  
  context.clearRect(0, 0, width, height); 
  context.beginPath(); 
  context.moveTo(0, height);  
  for (count = 0; count < length; count++) { 
    context.lineTo(count / length * width,  
        fft.spectrum[count] * -height * 2 + height); 
  }  
  context.moveTo(width,0); 
  context.closePath(); 
  context.fill(); 
  context.stroke(); 
}

You can use setTimeout() to continuously call the drawFFT() method. You can also use the Sink object, which is from the sink.js project and also bundled with audiolib.js:

Sink.doInterval( function(){   
  drawFFT();  
}, 1000/fps);

Cake! Now you can add some nice graphical output to your audiolib.js apps! fft.js can be used outside of audiolib.js too, if you have another use for it. Happy coding!
